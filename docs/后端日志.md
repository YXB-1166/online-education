# 后端开发日志 — 在线教育辅助教学系统

## 技术栈
- Spring Boot 3.2.0 + MyBatis + MySQL 8.0 + Nacos 2.3.0
- Maven groupId `com.edu`，JDK 21
- 多模块架构：edu-common、edu-user（8010）、edu-course（8020）、edu-exam（8030）、edu-gateway（8000）

## 数据库
- Docker 容器 `mysql-edu`，端口 3306，root/123456，字符集 utf8mb4
- `sql/init.sql`：7 张表（tb_user、tb_course、tb_course_selection、tb_assignment、tb_submission、tb_forum_post、tb_forum_reply）
- 中文数据导入：`docker cp` + 容器内 `mysql --default-character-set=utf8mb4`（PowerShell 管道损坏中文）
- `enrolled_count` 在 tb_course 中通过 `UPDATE ... COUNT(*)` 同步
- 容器重建后从旧卷 `c4b16fdf70abb` 恢复用户数据

## Maven 多模块
- 父 POM 管理依赖版本，5 个子模块
- `mysql-connector-j`（新 artifact，runtime scope）
- `jjwt-api/impl/jackson 0.12.3` — JWT 认证
- `springdoc-openapi-starter-webmvc-ui 2.3.0` — Swagger
- `spring-cloud-alibaba 2023.0.3.2` — Nacos 服务发现
- `nacos-client 2.4.2` — 显式添加解决 ClassNotFoundException（NacosLogging）

## 通用模块（edu-common）
- `Result<T>` — 统一响应体
- `BusinessException` — 业务异常
- `GlobalExceptionHandler` — 全局异常处理（需 `scanBasePackages` 包含 `com.edu.common`）
- `JwtUtil` — HMAC-SHA JWT 生成/解析（24h 过期）
- `JwtAuthFilter` — Servlet Filter，排除可配置路径，返回 401 JSON
- `@RequireRole` + `RoleInterceptor` — 角色注解拦截（返回 403 JSON）
- `PageParam` + `PageResult` — 手动分页（LIMIT/OFFSET，不用 PageHelper）
- `BaseService` — 抽象类提供日志 Logger

## 用户模块（edu-user）
- `POST /user/login` — JWT 登录返回 {code, data: {token, user}}
- `UserController.login` 添加 `@Value("${server.port}")` 日志输出（`当前服务端口号：{port}`）用于负载均衡演示
- 用户 CRUD、分页列表（角色 1/2/3 均可访问）
- 第二个实例端口 8011 验证 Nacos 负载均衡

## 课程模块（edu-course）
- 课程 CRUD、分页
- 选课审批流：
  - 学生选课 → status='0'（待审核）
  - 教师审批 → status='1'（已选课，enrolled_count+1）
  - 教师驳回 → status='4'（审核不通过）
  - 学生退选 → status='2'（已退选，enrolled_count-1）
  - 状态转换用 `and status = 'X'` 条件守卫
- `enrolled_count > 0` 防止负数
- `GET /course/selection/list?courseId=&teacherId=` — 教师查看课程学生名单（JOIN 限制只能看自己课程）

## 考试模块（edu-exam）
- 作业 CRUD
- 作业提交（tb_submission）
- 论坛帖子 + 回复

## 网关（edu-gateway）
- `spring-cloud-starter-gateway`（响应式，非 spring-boot-starter-web）
- `CorsConfig.java` — 全局 CORS
- `application.yml` — 路由使用 `lb://edu-user`、`lb://edu-course`、`lb://edu-exam`（Nacos 动态发现）
- 转发 Authorization 头

## Nacos 集成
- Nacos Server 2.3.0 安装于 `D:\ChineseSoftWork\nacos-server-2.3.0\nacos`
- 启动方式：`startup.cmd -m standalone`（端口 8848，admin/nacos）
- 4 个微服务全部注册到 Nacos
- Gateway 通过 `lb://` 实现客户端负载均衡
